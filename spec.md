# 技術指示書: AIパイプラインの次Issue生成エラー修正

## 概要

本指示書は、AIパイプラインにおける「ChatGPT - 次のタスクを生成してループする」工程で発生しているエラー `TypeError: unhashable type: 'dict'` の原因調査と修正を行うための手順を示します。

## タスク詳細

- タスク名: AIパイプラインの次Issue生成エラー修正
- エラー詳細: TypeError: unhashable type: 'dict'
- 発生箇所: 「ChatGPT - 次のタスクを生成してループする」工程

## 検討項目

以下の項目について確認および修正を行います。

1. **Pythonコード内で dict を set の中に入れていないか確認**
   - `dict` 型のオブジェクトが `set` に含まれていないかを確認します。`dict` は変更可能なタイプであり、`set` の要素としては使用できません。

2. **`{ {"key": "value"} }` のような記述がある場合の修正**
   - 発見した場合は、誤って `set` またはネストされた `dict` を使っていないか確認し、正しく `[ {"key": "value"} ]` のようにリストでラップするよう修正します。

3. **GitHub Issue作成用の `labels` / `assignees` / `body` / `title` の生成部分を確認**
   - `labels`、`assignees`、`body`、`title` の生成プロセス内でデータ型の誤りがないか確認し、適宜修正を行います。

4. **JSONデータは `dict`、複数データは `list` として扱う**
   - JSONデータは辞書型として、複数データセットや並列要素はリスト型として正しく扱っているか確認します。

5. **修正後、AIパイプラインが最後まで通ることを確認**
   - 修正後にパイプラインが正常に最後まで実行され、期待する結果が得られることを確認します。

## 完了条件

作業完了の判断基準は以下の通りです。

- `TypeError: unhashable type: 'dict'` エラーが解消される。
- 「次のタスクを生成してループする」工程が問題なく完了する。
- GitHub Issueの作成または次タスクの生成が正常に行われることを確認する。

- エラーの調査に必要な情報は、システムログおよび関連するコードレビューから取得してください。

- 修正においてはコードの可読性や保守性についても考慮して行ってください。
