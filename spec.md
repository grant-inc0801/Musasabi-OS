# 技術指示書: Sprint 3 定義 — Sales Learning MVP

## 1. 概要

この技術指示書では、Sales Learning MVPを実現するためのSprint 3定義について説明します。このスプリントでは、テレマーケティング営業部門向けの最速で使用可能なSales Learning MVPの構築に焦点を当てています。スプリント定義ファイルはAI PMの基準となります。

## 2. ファイル構成

### 作成するファイル

- `docs/sprints/Sprint-003.yaml`

## 3. スプリントメタデータ

- `sprint_key`: S3
- `title`: Sales Learning MVP
- `status`: planned
- `priority`: critical
- `owner`: CEO
- `maintainer`: Musasabi CTO

## 4. スプリントゴール

社内の最速で使用可能なセールスラーニングシステムを構築します。営業部門は以下のタスクを実行できる必要があります:

- コール情報のインポートまたは手動登録
- 文字起こしとメモの追加
- 営業コールの分析
- 異議と反論のキャプチャ
- MUSAコーチングの受講
- セールス知識の蓄積

## 5. タスク詳細

### S3-004-01: Call Transcript + Manual Notes

- **目的**: ユーザーがコールの文字起こしを手動で作成・編集できるようにする。
- **依存関係**: なし
- **受け入れ基準**:
  - 文字起こしテーブルが存在すること
  - 文字起こしエディタが機能すること
  - リードに文字起こしがリンクされていること
  - コールログに文字起こしがリンクされていること
  - 文字起こしのサマリが機能すること

### S3-004-02: Sales Call Analysis Scoring

- **目的**: 文字起こしを分析し、営業スコアを生成する。
- **依存関係**: S3-004-01
- **受け入れ基準**:
  - トーンスコア
  - リアクションスコア
  - リスニングスコア
  - 質問スコア
  - クロージングスコア
  - 総合スコア

### S3-004-03: Objection and Rebuttal Learning

- **目的**: 異議を検出し、推奨される反論を保存する。
- **依存関係**: S3-004-01
- **受け入れ基準**:
  - 異議のパターンテーブルが存在すること
  - 反論記録を保存できること
  - 成功/失敗カウントを更新できること

### S3-004-04: MUSA Sales Coaching Panel

- **目的**: 各コール文字起こし後にコーチングの提案を表示する。
- **依存関係**: S3-004-02, S3-004-03
- **受け入れ基準**:
  - 強みが表示されること
  - 弱みが表示されること
  - 改善点が表示されること
  - 推奨スクリプトが表示されること
  - 次のコールへのアドバイスが表示されること

### S3-004-05: Sales Knowledge Candidate Creation

- **目的**: 有用なコールの洞察から知識候補を作成する。
- **依存関係**: S3-004-02, S3-004-03
- **受け入れ基準**:
  - 知識候補を作成できること
  - 候補はソースの文字起こしを含むこと
  - 候補は公開前に人間の承認が必要

### S3-004-06: Sales Learning Dashboard

- **目的**: 営業チームの学習進捗を表示する。
- **依存関係**: S3-004-02, S3-004-03, S3-004-05
- **受け入れ基準**:
  - 今日のコール数
  - 平均スコア
  - トップ異議
  - トップ反論
  - 知識候補
  - コーチング回数
