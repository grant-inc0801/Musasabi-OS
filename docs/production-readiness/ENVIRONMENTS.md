# 環境分離 設計テンプレート(dev / staging / prod)

> 設計のみ。実環境の構築・接続は Production Readiness の人間承認後に実施する。

## 3環境の分離
| 環境 | 目的 | データ | secrets | アクセス |
|------|------|--------|---------|----------|
| dev | 開発・Mock 検証 | ダミー/Mock | ダミー | 開発者 |
| staging | 本番相当の検証・負荷試験 | マスキング済み | staging 用(分離) | 限定 |
| prod | 本番 | 実データ | prod 用(分離・最小権限) | 承認された運用者のみ |

## 原則
- 環境ごとに設定・secrets・アクセス制御を**完全分離**する。
- prod の secrets は staging/dev から参照不可。
- 変更は dev → staging → prod の順に、承認ゲートを通す。

## バックアップ/DR(概要)
- prod は定期自動バックアップ+暗号化保管。RPO/RTO を定義。DR リージョンを用意。
- リストア手順を文書化し、定期的に復旧演習を行う。

## データ移行(概要)
- 前方互換マイグレーション + ドライラン + 事前バックアップ + ロールバック計画。
- 実データ移行は承認後にのみ実施する。
