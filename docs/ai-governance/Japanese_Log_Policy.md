# Musasabi OS 日本語ログ運用ポリシー

## 目的
Musasabi OSでは、開発・運用・ハンドオフの可読性を高めるため、日本語を標準表示言語とする。

Claude Codeは、ユーザー向け・開発者向けの報告、実行ログ、GitHub上の説明、handoff docsを原則として日本語で記述する。

## 日本語で記述する対象

以下はすべて日本語で出力・記述すること。

- Claude Codeの実行ログ
- Claude Codeが生成する進捗メッセージ
- GitHub Issue本文
- GitHub PR本文
- GitHub PRレビューコメント
- `docs/ai-handoff/*`
- `docs/**/*.md`
- エラー説明
- 警告説明
- テスト結果の要約
- 進捗報告
- リリースノート
- CHANGELOGの説明文
- ChatGPTへの質問
- Claude Codeからの確認事項

## 英語のままでよい対象

以下は英語のままでよい。

- ソースコード
- クラス名
- 関数名
- 変数名
- ファイル名
- ディレクトリ名
- API名
- ライブラリ名
- Gitコマンド
- npmコマンド
- Rust / Tauri標準メッセージ
- 外部ツールが自動出力する英文ログ

## 実行ログ例

悪い例:

```text
Running tests...
Building packages...
Updating architecture...
Finished.
```

良い例:

```text
テストを実行しています…
パッケージをビルドしています…
アーキテクチャを更新しています…
処理が完了しました。
```

## テスト結果フォーマット

```markdown
## テスト結果

成功：122件
失敗：0件
スキップ：0件

すべてのテストに成功しました。
```

## エラー報告フォーマット

```markdown
## エラー内容

## 原因

## 影響範囲

## 対処内容

## 次の対応
```

## 実装完了報告フォーマット

```markdown
━━━━━━━━━━━━━━

━━━━━━━━━━━━━━
```

## Debugログ

デバッグ目的のログには `[DEBUG]` を付ける。

```text
[DEBUG] 設定ファイルの読み込み結果を確認しています。
```

## 重要ルール

- Musasabi OSの標準表示言語は日本語とする。
- コード識別子は英語のままでよい。
- 技術的な正確性を優先し、日本語化によって意味が曖昧になる場合は英語名を併記する。
- 外部ツール由来の英語ログは無理に改変しなくてよいが、Claude Codeの要約は日本語で行う。
