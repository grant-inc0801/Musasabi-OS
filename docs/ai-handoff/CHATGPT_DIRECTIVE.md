# ChatGPT Directive

## Directive ID
D-20260706-004

## Title
管理画面UIとデスクトップ右下アバター常駐動作をβ版に反映する

## 決定
管理画面は、ユーザーが提示した画像イメージのようなダークテーマのダッシュボードUIを基本方針とします。

アプリを縮小・最小化した場合は、メイン管理画面を隠し、アバターだけをデスクトップ右下に常駐表示してください。

アバターをクリックすると、モード切り替えとチャット欄を含むミニパネルを表示します。必要に応じて、吹き出しで提案・通知・次アクションを表示できるようにしてください。

## 目的
β版では、ユーザーが以下の体験を確認できることを優先します。

- 管理画面で主要機能を操作できる
- 画面を縮小するとアバターだけが右下に残る
- アバターをクリックするとミニ操作パネルが出る
- Learning / Test / AutoCall のモード切り替えができる
- チャット欄からAI社員へ指示できる
- 吹き出しで提案・通知を表示できる

## 実装指示

1. 管理画面UIはダークテーマのβ版ダッシュボードとして整える。
2. 画面縮小・最小化時に、メインウィンドウを隠して右下アバターのみ表示する挙動を設計・実装する。
3. Tauriで可能な範囲で、常駐アバター用の小型ウィンドウまたは常時表示オーバーレイを用意する。
4. アバターは現時点では白黒ムササビアイコン、または2Dプレースホルダーでよい。
5. アバタークリック時にミニパネルを表示する。
6. ミニパネルには以下を含める。
   - 現在のモード表示
   - Learning / Test / AutoCall のモード切り替え
   - チャット入力欄
   - 送信ボタン
   - 最新提案または通知の表示
7. 吹き出しUIを追加し、提案・通知・注意事項を短文表示できるようにする。
8. AutoCall本番実行は無効のまま維持し、AutoCallは準備中または承認待ちとして表示する。
9. 実架電・実API接続・実認証情報保存は行わない。
10. β版READMEに、管理画面と右下アバター操作方法を追記する。
11. Windows実機確認チェックリストに、最小化・右下常駐・アバタークリック・ミニパネル表示を追加する。
12. テスト可能な範囲でUI状態管理のテストを追加・更新する。
13. 結果を日本語で `docs/ai-handoff/CLAUDE_RESPONSE.md` に記録する。

## UI仕様

### 管理画面
- ダークテーマ
- サイドバー
- ダッシュボードカード
- AI社員ステータス
- モード状態
- Settings導線
- Learning / Test / AutoCall への導線

### 右下アバター
- デスクトップ右下に表示
- 小さな常駐アイコンとして表示
- クリック可能
- メイン画面を開く導線を持つ
- ミニパネルを開く導線を持つ

### ミニパネル
- モード切替
- チャット欄
- 提案表示
- メイン管理画面を開くボタン

### 吹き出し
- 例：「次はTest Modeでロールプレイ確認しましょう」
- 例：「AutoCallは承認待ちです」
- 例：「Learning内容をSales Brainへ反映しました」
