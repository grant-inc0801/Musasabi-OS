# ChatGPT Directive

## Directive ID
D-20260706-005

## Title
β版確認後の修正：部門サイドバー・部門別ページ・右下アバター表示・Learning作業学習

## 決定
ユーザーがβ版を確認した結果、次の修正を優先してください。

1. 左サイドバーは機能名ではなく、まず部門名を表示する。
2. 各部門を選択すると、その部門に割り当てられたAI社員、成果物、KPI、業務ページを表示する。
3. 営業部はコールシステム、営業ダッシュボード、Learning / Test / AutoCall モード切替を中心にする。
4. 出版部は成果物、販売数、各成果物の売上、出版進捗を中心にする。
5. 画面最小化時にアバター以外の透明ウィンドウや余計な表示を出さない。
6. アバタークリック時のみ、入力欄とモード切替パネルを表示する。
7. Learning Modeでは、資料だけでなく日々の作業内容も学習できるようにする。

## 背景
β版の管理画面方向性は概ねOKです。ただし、Musasabi OSは単なる機能メニューではなく、AI社員と部門で動くAI会社OSです。

そのため、メニュー構造は「営業AI」「設定」などの機能一覧ではなく、まず「営業部」「出版部」などの部門単位で整理してください。

## UI修正指示

### 1. 左サイドバー
左サイドバーには部門名を表示してください。

初期部門例：

- 営業部
- 出版部
- AI企画部
- 管理部
- 設定

各部門をクリックすると、中央エリアにその部門のページを表示します。

### 2. 部門ページ共通構成
各部門ページには以下を表示してください。

- 部門名
- 部門概要
- 割当AI社員
- 現在のモード
- 成果物 / タスク / KPI
- 部門別ダッシュボード
- 部門別アクション

### 3. 営業部ページ
営業部では以下を中心に表示してください。

- コールシステム
- 営業ダッシュボード
- Learning Mode
- Test Mode
- AutoCall Mode（本番実行不可）
- Sales Brain
- テストコール履歴
- 改善提案
- AI営業社員一覧

### 4. 出版部ページ
出版部では以下を中心に表示してください。

- 成果物一覧
- 販売数
- 各成果物の売上
- 執筆 / 編集 / 校正 / 表紙 / 出版ステータス
- AI編集長 / AI著者 / AI校正担当
- Kindle / note 等の販売先はMockまたは準備中表示

### 5. その他部門
AI企画部、管理部などは現時点ではMockページで構いません。

ただし、将来的に部門を追加できるデータ構造にしてください。

## 右下アバター修正指示

### 問題
画面最小化時に、アバター以外の透明な画面が表示されています。

### 修正方針
最小化時は、アバター以外を表示しないでください。

実装要件：

- メイン管理画面を最小化・非表示にする
- デスクトップ右下にはアバターのみ表示する
- 透明背景ウィンドウの余白、不可視領域、クリックを妨げる領域を残さない
- アバタークリック時のみミニパネルを表示する
- ミニパネルを閉じたら、再びアバターのみ表示に戻る

## アバタークリック時のミニパネル

アバタークリック時のみ、以下を表示してください。

- チャット入力欄
- 送信ボタン
- 現在のモード
- Learning / Test / AutoCall のモード切替
- メイン管理画面を開くボタン
- 最新提案または吹き出し

通常時はアバターだけで、入力欄やモード切替は表示しないでください。

## Learning Mode 修正指示

Learning Modeでは、アップロード資料だけでなく、日々の作業内容も学習できるようにしてください。

### 学習対象

- 日々の作業内容
- 操作履歴
- 入力内容
- 完了タスク
- 保留タスク
- 会話ログ
- テストコール結果
- 改善指摘
- 部門別成果物

### β版での実装範囲

現フェーズでは、実OS操作の完全監視までは不要です。

まずは以下で十分です。

- 作業メモ入力欄
- 今日の作業内容追加
- 完了 / 保留 / 改善メモ
- 部門選択
- AI社員への学習反映Mock
- 学習履歴一覧

将来的には、ユーザー承認のもとで操作履歴学習へ拡張してください。

## 安全ルール

Claude Codeは以下を行わないこと。

- 実電話番号への発信
- AutoCall本番実行の有効化
- FileMaker本番DBへの書き込み
- Zoom Phone本番APIへの接続
- 実認証情報の保存
- ユーザー操作の無断監視
- secretsの表示
- force push

Learning Modeで作業内容を学習する場合も、必ずユーザーが明示的に入力・登録した内容を対象にしてください。
