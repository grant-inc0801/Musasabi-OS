# ChatGPT Directive

## Directive ID
D-20260708-001

## Title
現フェーズ完成・部門拡張・アバター/デスクトップ化をノンストップで進行する

## 決定
Musasabi OS は、現フェーズの完成を最優先に進めてください。

現在の最重要方針は以下です。

1. 現フェーズの未完了タスクを完了させる
2. 承認が不要な実装・修正・ドキュメント更新は停止せず進める
3. 保留になっているタスクを再確認し、実装可能なものは完了へ進める
4. 部門機能を拡張する
5. 特に営業部門と出版部門を優先して整備する
6. アバター機能を並行して進める
7. デスクトップアプリ化を並行して進める
8. Claude Code から質問・選択肢が出た場合は、ChatGPT の方針に沿って実装指示へ変換できる状態を維持する

このDirectiveは、Claude Code / Codex / GitHub Actions / AI PM が迷わず次の実装へ進むための最新指示です。

Musasabi OS を「社内で実際に使い始められる最小稼働版」に近づけることを最優先にしてください。

現時点では、完璧な商用版よりも以下を優先します。

- 画面が起動する
- 主要パネルが表示される
- 部門状態が見える
- AI社員/部門の役割が確認できる
- Mockデータで業務フローが動く
- アバター吹き出しが状態を要約できる
- 承認・保管庫・営業・出版・企画・開発の流れが見える
- README / CLAUDE_RESPONSE.md / テストが更新される

---

## 2. ノンストップ実装ルール

Claude Code / Codex は、以下に該当する場合はユーザー確認で停止しないでください。

- UI文言の調整
- Mockデータ追加
- テスト追加
- README更新
- CLAUDE_RESPONSE.md更新
- 既存設計に沿った型追加
- 既存UIに沿ったカード/パネル追加
- 安全なリファクタリング
- lint/test修正

- 実課金が発生する場合
- 実ユーザーデータを読み書きする場合

- force push が必要になる場合
- 破壊的変更が必要になる場合
- 既存仕様と明確に衝突する場合

## 4. 部門機能の基本方針

Musasabi OS には「AI社員が所属する部署/部門」という概念を明確に入れてください。

各部門は以下を持ちます。

- 部門名
- 目的
- 担当AI社員
- 現在の稼働状態
- 進行中タスク
- 完了タスク
- 保留タスク
- 次の推奨アクション
- 関連ドキュメント
- 保管庫連携状態

初期部門例：

- 経営部門
- 企画部門
- 開発部門
- 営業部門
- 出版部門
- 管理部門
- 市場調査部門
- カスタマーサポート部門
- 保管庫管理部門

## 5. 営業部門を優先実装する

営業部門は、Musasabi OS の収益化・実務活用に直結するため優先してください。

営業部門の役割：

- 営業リスト管理
- 架電対象の整理
- トークスクリプト管理
- 切り返し集の管理
- アポ獲得状況の管理
- 営業資料の作成依頼
- 企画部門との連携
- 保管庫への営業資料保存
- 日次営業レポート作成

Mock実装でよいので以下を表示してください。

- 本日の架電予定
- 本日のアポ目標
- 獲得アポ数
- 改善対象トーク
- よくある反論
- 推奨切り返し
- 営業資料作成依頼
- 次の営業アクション

将来のZoom Phone / FileMaker連携は後続フェーズ扱いとし、現フェーズでは実接続しないでください。

## 6. 出版部門を優先実装する

出版部門は、Claude Code等で作成したライトノベル・書籍・note記事などを半自動で商品化するための部門です。

出版部門の役割：

- 原稿企画
- 作品管理
- 校正確認
- 既存作品との類似性チェック方針管理
- 著作権/権利確認チェックリスト管理
- Kindle出版準備
- note販売準備
- 表紙/紹介文/販売文の作成依頼
- 編集長AIのタスク管理
- 保管庫への原稿・校正履歴保存

Mock実装でよいので以下を表示してください。

- 企画中作品
- 執筆中作品
- 校正待ち作品
- 類似性チェック待ち作品
- Kindle準備中作品
- note販売準備中作品
- 編集長AIからの指摘
- 次の出版アクション

現フェーズでは外部出版サービスへの実ログイン・実投稿・実販売設定は行わないでください。

---

## 7. 敏腕編集長AIの追加

出版部門には「敏腕編集長AI」を追加してください。

役割：

- 原稿の改善点を指摘
- 物語構成を確認
- キャラクター設定の一貫性を確認
- 既存作品と酷似しないための注意点を提示
- 読者ターゲットに合わせた改善提案
- タイトル案/帯文/紹介文を作成
- Kindle/note向け販売導線を提案

表示例：

```text
敏腕編集長AI：第1章は導入として良好です。ただし主人公の目的が弱いため、冒頭3ページ以内に明確な動機を追加してください。
```

## 8. アバター機能の並行実装

Musasabi OS の標準アバターはムササビです。

アバター方針：

- デフォルトはムササビ
- 右下常駐を想定
- 音声なし
- 吹き出しテキストで状態表示
- モード切替UIを持つ
- 部門状態を要約する
- 重要タスク・承認待ち・保管庫容量・営業進捗・出版進捗を知らせる

現フェーズでは3D実装にこだわらず、静的/簡易アニメーション/Mock表示でよいです。

表示例：

```text
ムササビAI：営業部門は本日20件の架電予定があります。出版部門では1件の原稿が校正待ちです。
```

## 9. デスクトップアプリ化の並行実装

Musasabi OS は最終的にPC上で常駐するデスクトップアプリとして動かします。

現フェーズでは以下を優先してください。

- 既存の apps/desktop を安定起動させる
- 依存を増やしすぎない
- npm run dev:desktop で確認できる状態を維持する
- UIパネル追加を優先する
- Tauri等の本格統合は後続フェーズでもよい
- Windows 11での利用を前提にする

必ずテストを追加/更新してください。

## 10. 保管庫との連携

保管庫は以下の部門成果物を保存するMockフローを持ちます。

- 企画部門が作成した提案資料
- 営業部門が利用する営業資料
- 出版部門が作成した原稿/校正履歴
- 開発部門が作成した仕様書
- 管理部門が作成した運用資料

実ファイル保存・実クラウド連携・実ダウンロードはまだ行わないでください。

## 11. ChatGPT / Claude Code 連携ルール

Claude Code から質問や選択肢が提示された場合、次のルールで処理してください。

- 既存Directiveに沿うなら、最も安全で実装が進む選択肢を採用する
- ユーザー承認が不要なものは停止しない
- 迷う場合はMock実装を優先する
- 外部接続や実データ操作が必要な場合のみ保留する
- 判断内容は CLAUDE_RESPONSE.md に日本語で記録する

Claude Codeへの返答形式は以下を推奨します。

```text
方針：〇〇で進めてください。
理由：現フェーズでは安全性と最小稼働を優先するためです。
実装：Mockデータ/UI/テスト/README/CLAUDE_RESPONSE.mdを更新してください。
停止条件：secrets、実課金、実データ操作、外部接続が必要な場合のみ停止してください。
```
