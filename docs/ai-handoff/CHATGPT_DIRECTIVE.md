## 画面レイアウト

### 基本構造

画面は大きく以下の4領域に分けてください。

1. 左サイドバー
2. 中央：部署パネル一覧
3. 右：選択部署の詳細パネル
4. 下：部署指定チャット指示欄

右下にはアバターを常駐表示します。

## 1. 左サイドバー仕様

左サイドバーは細めにし、表示要素を最小限にしてください。

表示するもの：

- アプリアイコン
- Musasabi OS ロゴ名
- 全社員数
- 稼働率
- 設定ボタン

表示しないもの：

- 部署メニュー一覧
- 細かい機能メニュー
- 不要なナビゲーション

サイドバーのイメージ：

```text
[アプリアイコン]
Musasabi OS

全社員数
28人
（稼働中：24人）

稼働率
85%

[設定]
```

## 2. 中央：部署パネル一覧

中央には部署パネルだけを配置してください。

部署パネルに表示する内容は最小限にします。

表示項目：

- 部署名
- 社員数
- ステータス

表示しないもの：

- 詳細な作業内容
- 長文説明
- 売上詳細
- ログ

それらは右側詳細パネルに表示します。

### 部署パネル例

```text
営業部
8人
● 完了
```

```text
出版部
3人
● 承認待ち
```

```text
システム開発部
6人
● エラー
```

### 初期部署例

- 営業部
- カスタマーサポート部
- マーケティング部
- システム開発部
- 出版部
- 企画部
- 経理部
- 人事部

## 3. ステータス色分け

ステータスごとに、ランプ色とパネル枠色を必ず統一してください。

- 完了：緑 `#22C55E`
- 作業中：黄色 `#FACC15`
- エラー：赤 `#EF4444`
- 承認待ち：紫 `#A855F7`

### ランプ仕様

- 小さな丸ランプ
- 少し発光しているように見せる
- パネル下部またはステータス横に配置

### 枠仕様

- ステータスと同じ色で発光
- glow / box-shadow を付ける
- 通常時も薄く点灯
- ホバー時は少し強く光る

## 4. 部署間連携ライン

部署間で連携している場合は、パネル同士を白い発光ラインでつないでください。

仕様：

- 白色 `#FFFFFF`
- 半透明
- glowあり
- パネル間の中心を結ぶ線
- ライン上に小さな白いノード点を配置
- 連携中はラインがほのかに流れるようなアニメーション

例：

- 営業部 → カスタマーサポート部
- カスタマーサポート部 → マーケティング部
- 出版部 → 企画部
- 企画部 → 経理部
- システム開発部 → 人事部

β版ではMockデータで構いません。

## 5. 右側：部署詳細パネル

部署パネルをクリックすると、右側に該当部署の詳細を表示してください。

表示項目：

- 部署名
- ステータス
- 社員数
- 作業内容
- 作業進捗
- 作業ログ一部
- 部署固有のKPI

### 営業部パネル内の専用表示

営業部を選択した場合は、コールシステム関係を配置してください。

表示項目：

- コールシステム
- 架電数（本日）
- 接続率
- アポ獲得数
- 成約数
- コール状況
- 待機中件数
- 通話中件数
- 後処理中件数
- エラー件数
- 作業ログ
- 詳細を見るボタン

実架電は絶対に行わず、Mock表示のみです。

## 6. 下部：部署指定チャット指示欄

画面下部に横長のチャット欄を設けてください。

目的：ユーザーが部署を選んで直接指示を出せるようにする。

### 必須UI

- 指示先部署プルダウン
- メッセージ入力欄
- 送信ボタン
- 音声入力ボタン
- ファイル添付ボタン
- クイック指示テンプレート

### 表示例

```text
指示先部署 [営業部 ▼]
[メッセージを入力してください...]
[ファイル添付] [音声入力（押して話す）] [送信]
```

### クイック指示テンプレート例

- 進捗状況を教えて
- 本日の目標を共有して
- 課題を報告して
- 優先タスクを教えて

### 音声入力

β版では実音声認識はMockで構いません。

- 音声入力ボタンを表示
- 押下時に「音声入力準備中」またはMock録音状態を表示
- 実STT接続は後続フェーズ

## 7. 右下アバター仕様

今回のアバターは、画像イメージに近い「可愛い3D風ムササビ」にしてください。

### 見た目

- 小さくて可愛い3Dマスコット風
- ムササビ / リス寄りの丸い顔
- 大きな黒い目
- 小さな鼻
- ふわっとした耳
- グレー〜シルバー系の毛色
- 黒またはダークグレーのパーカー
- 胸元に小さなムササビロゴ
- ゴーグルまたは近未来アクセサリー
- 右下に常駐

### 表情

- 通常：にこやか
- 承認待ち：少し考え中
- エラー：困り顔
- 完了：嬉しそう

β版では静止画像・CSS・SVG・PNGプレースホルダーでも構いませんが、見た目はこの方向で寄せてください。

### 吹き出し

アバターは吹き出しで現状を要約してください。

特に以下を伝える：

- 承認待ちがどの部署にあるか
- エラーがどこで起きているか
- なぜエラーが起きたか
- どう解決するか
- 全体の進行状況

吹き出し例：

```text
現在、出版部と人事部で承認待ちがあります。
システム開発部ではAPI接続エラーが発生しています。
原因は外部APIの認証期限切れの可能性があります。
APIキーを更新し、再接続してください。
```
